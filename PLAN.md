# План: Telegram-бот для волейбольной команды «Иппон»

## 1. Общая концепция

Бот автоматизирует две рутинные задачи команды:
- **Финансы:** парсит Google Таблицу с арендой зала и периодически напоминает должникам.
- **Посещаемость:** автоматически создаёт опросы перед тренировками и напоминает тем, кто ещё не отметился.

Принцип: минимум инфраструктуры, максимум использования существующих привычек команды (Google Sheets, Telegram-группа).

---

## 2. Работа с нативными опросами

> **Важное уточнение:** Если **бот сам создаёт нативный неанонимный опрос** (`is_anonymous=False`), Bot API присылает события `PollAnswer`, которые содержат `user` (кто голосовал) и `option_ids` (за какой вариант).
>
> Это означает:
> - ✅ Участники видят нативный опрос и могут видеть, кто как проголосовал
> - ✅ Бот получает полную информацию о голосах и может напоминать тем, кто ещё не проголосовал
> - ✅ Всё в рамках стандартного Bot API, без костылей

### Алгоритм работы с опросами
1. Бот публикует **неанонимный** нативный опрос (poll) в группу (в тему «Опросы», thread_id=2).
2. Бот слушает события `PollAnswer` и записывает голоса в лист «Голоса».
3. За день до тренировки бот напоминает в группе тем, кто ещё не проголосовал.
4. Результаты хранятся в Google Таблице.

## 3. Ответы на технические вопросы

### 3.1 Хостинг на старом ноутбуке (минимальный вред)

**Аппаратные рекомендации:**
- Убрать аккумулятор (если он старый и сильно греется) или оставить с ограничением заряда 60-80% в BIOS.
- Поднять заднюю часть ноутбука для лучшей конвекции.
- Закрыть крышку — ОК, но в настройках энергосбережения отключить сон при закрытии крышки.

**Software:**
- **Polling, а не Webhook.** Webhook требует белого IP + проброс портов + TLS. Polling проще и надёжнее для домашнего хостинга.
- **Systemd юнит** для автозапуска бота при загрузке и автоперезапуска при падении. Manjaro использует systemd, конфиг будет универсальным.
- **Python venv** — изолированное окружение.
- **Logrotate** — чтобы логи не забивали диск.
- **Аптайм:** бот практически не грузит CPU (только раз в N минут делает запросы). Нагрузка сопоставима с просто включённым браузером.

### 3.2 Git и раскатка релизов

**Репозиторий:** публичный или приватный на GitHub / GitLab.

**Простейший flow (рекомендуется для старта):**
```
main          → продакшен (стабильный код, запущен на ноутбуке)
develop       → интеграция фич
feature/xxx   → отдельные задачи
```

**Деплой (manual, KISS):**
1. На ноутбуке клонируется репозиторий.
2. Для обновления: `git pull origin main && sudo systemctl restart ippon-bot`
3. Можно обернуть в простой bash-скрипт `deploy.sh`.

**Без CI/CD pipline на старте.** Это лишняя сложность для одного ноутбука и двух разработчиков. Если проект вырастет — перейдём на GitHub Actions + webhook для автодеплоя.

### 3.3 Два бота: dev и prod

Да, нужно два бота. Регистрируем обоих через @BotFather:
- `@ippon_volley_bot` — продакшен (в рабочей группе)
- `@ippon_volley_test_bot` — для разработки и тестов (в тестовой группе или личке)

**Реализация:**
- Один кодовый репозиторий.
- Разные `.env` файлы: `.env.prod` и `.env.dev`.
- Запуск через `python -m bot` с указанием конфига (`--env-file .env.dev`).
- На ноутбуке крутится только `prod`. `dev` запускается локально на машине разработчика.

### 4.3 Потоки данных

**Напоминания должникам:**
```
APScheduler (раз в неделю)
  → google_sheets.get_debtors() → gspread → Google Sheets (J2:K15)
  → Фильтр: balance < -20
  → Попытка отправить ЛС каждому (если бот может)
  → Оставшихся упоминаем в группе (General)
```

**Создание опроса перед тренировкой:**
```
APScheduler (ежедневно в 10:00)
  → training_scheduler.check_trainings()
  → Если за 2 дня до тренировки:
    → poll_service.create_training_poll() → bot.send_poll() → группа (thread_id=2)
    → google_sheets.save_poll() → лист "Опросы"
```

**Голосование:**
```
Пользователь голосует в опросе
  → handlers/polls.py → PollAnswer
  → google_sheets.save_vote() → лист "Голоса"
```

**Напоминание не проголосовавшим:**
```
APScheduler (за 1 день до тренировки)
  → training_scheduler.check_trainings()
  → poll_service.remind_non_voters()
  → Сверяем members.json с листом "Голоса"
  → Отправляем сообщение в группу (thread_id=2)
```

### 4.4 Где хранить состояние без БД

| Данные | Хранилище |
|--------|-----------|
| Долги, оплаты | Лист «Долги» (J2:K15): имя, баланс |
| Опросы | Лист «Опросы»: poll_id, message_id, date, time, location, thread_id, status |
| Голоса | Лист «Голоса»: poll_id, имя, выбор, время голосования |
| Список участников | `data/members.json`: имя → user_id |
| Расписание тренировок | `data/trainings.json` |
| Шаблоны сообщений | `templates/*.txt` |
